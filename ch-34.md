# Ch-34 이터러블

## 이터러블 프로토콜
- 이터러블 프로토콜은 반복 가능한 객체를 만들기 위한 규칙
- 이터러블 프로토콜을 따르는 객체는 for...of 문과 전개 연산자, 나머지 매개변수 등에서 반복 가능한 객체로 취급
- 이터러블 프로토콜을 따르는 객체는 Symbol.iterator 메서드를 구현해야 함
- Symbol.iterator 메서드는 이터레이터를 반환
- 이터레이터는 next 메서드를 구현해야 함
- next 메서드는 이터레이터가 순회할 수 있는 값을 반환
- next 메서드는 { value, done } 형태의 객체를 반환   

---

## 이터레이터 프로토콜
- 이터레이터 프로토콜은 이터레이터를 만들기 위한 규칙
- 이터레이터는 next 메서드를 구현해야 함
- next 메서드는 이터레이터가 순회할 수 있는 값을 반환
- next 메서드는 { value, done } 형태의 객체를 반환
- value는 이터레이터가 순회할 수 있는 값
- done은 이터레이터가 순회를 완료했는지 여부
- done이 true이면 이터레이터가 순회를 완료한 것
- done이 false이면 이터레이터가 순회를 완료하지 않은 것
- 이터레이터는 next 메서드를 호출할 때마다 value를 변경
- 이터레이터는 next 메서드를 호출할 때마다 value를 반환

---

## 이터레이션 프로토콜의 필요성
- 데이터 소비자와 데이터 공급자를 연결하는 인터페이스
- 데이터 소비자는 이터레이터를 통해 데이터를 소비
- 데이터 공급자는 이터레이터를 통해 데이터를 공급
- 데이터 소비자는 이터레이터를 통해 데이터를 소비할 때마다 next 메서드를 호출
- 데이터 공급자는 이터레이터를 통해 데이터를 공급할 때마다 next 메서드를 호출
- 데이터 소비자는 이터레이터를 통해 데이터를 소비할 때마다 value를 반환
